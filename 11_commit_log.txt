Commit Log :
1. Git Adalah Distributed Version Control, Artinya Walapun Kita Repository Di Local Komputer Kita,
   Semua Riwayat Perubahan Disimpan Di Komputer Kita
2. Kekurangannya Menjadi Makin Lama Repository Akan Semakin Besar Ukurannya,
   Namun Keuntungannya, Kita Bisa Melihat Semua Riwayat Commit Atau Disebut Commit Log
3. Untuk Melihat Commit Log, Kita Bisa Gunakan Perintah :
   git log

Log Sederhana :
1. Kadang Kita Hanya Ingin Melihat Commit Log Messagenya / Deskripsi Saja Atau Istilahnya Adalah Versi Sederhana Saja
2. Untuk Melakukan Itu, Kita Bisa Gunakan Perintah :
   git log --oneline

Kode Git Log --OneLine :
git log --oneline

723843c (HEAD -> master) Menghapus file3.txt
2868810 Melakukan Perubahan file1.txt 
Dan file2.txt
b62e3a6 Melakukan Perubahan file3.txt 
d3f3acf Menambah file2.txt Dan file3.txt
e713386 Menambah file1.txt

Graph :
1. Saat Nanti Kita Sudah Belajar Tentang Git Branching, Kadang Kita Ingin Melihat Commit Log Dengan Hubungannya Dengan Commit Log Sebelumnya
2. Hal Ini Bisa Kita Lakukan Menggunakan Perintah :
   git log --oneline --graph

Kode Git Log --OneLine --Graph :
git log --oneline --graph

* 723843c (HEAD -> master) Menghapus file3.txt
* 2868810 Melakukan Perubahan file1.txt Dan file2.txt
* b62e3a6 Melakukan Perubahan file3.txt
* d3f3acf Menambah file2.txt Dan file3.txt
* e713386 Menambah file1.txt

=> Kalau Menggunakan Git Graph Kompleks, Dia Akan Bercabang Masuk Ke Utama Karena Harus Menggunakan Branching

Melihat Detail Commit :
1. Kadang Kita Ingin Melihat Detail Perubahan Yang Terjadi Pada Sebuah Commit
2. Untuk Melakukan Itu, Kita Bisa Gunakan Perintah :
   git show 723843c / Hash
